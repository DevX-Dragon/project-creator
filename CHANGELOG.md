# Changelog

All notable changes to this project will be documented in this file.

## Why this file exists

- It gives users a quick way to see what changed between releases.
- It helps contributors understand the project history without digging through commits.
- It makes release notes easier to write and keeps them consistent over time.
- It is useful for debugging, because you can match a behavior change to the version that introduced it.

## [v3.0.1] 2026-04-29

### Added
- Expanded documentation with a full software guide in `docs/Documentation.md`
- Added a polished documentation landing page in `docs/index.html`
- Added `CONTRIBUTING.md`
- Added bundled starter templates under `templates/`
- Added support for loading templates from the GitHub contents API
- Added external source template loading via `source:https://...`

### Changed
- Reworked the main README into a clearer project overview
- Improved the CLI flow and Git handling
- Updated the generated project structure to include `.env.example`, `blueprint.json`, and `.gitignore`
- Updated the default Git branch flow to use `main`
- Marked the old GUI version as discontinued

### Fixed
- Improved template and release documentation consistency
- Cleaned up older CLI behavior and output messaging
